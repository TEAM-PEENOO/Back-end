# 최종 확정 기획안 — AI 협업 리포트 (사용자 아이디어 통합·보완)

> **참조 문서**: `initial_idea.md`, `initial_idea_2.md`
> **이 문서의 목적**: 사용자가 제안한 "나만의 학생 키우기" 아이디어를 비판적으로 평가하고,
> TeachBack AI 기반 아이디어와 통합해 최종 기획안을 확정한다.

---

## PART 1 — 사용자 제안 아이디어 비판적 평가

### 제안 요약
> 게임처럼 나만의 AI 학생 페르소나를 생성 → 초1부터 대학원생까지 레벨 성장 →
> 사용자가 지식을 가르치면 페르소나가 학습 → 페르소나 레벨에 맞는 시험을 함께 응시 →
> 합산 점수가 기준 이상이면 페르소나가 다음 학년으로 승급 →
> 페르소나는 인간처럼 망각 곡선을 따라 일부 내용을 잊어버림

---

### 강점 평가 ✅

#### 1. 교육학 원리와 게임 심리학의 절묘한 교차점
기존 TeachBack AI의 "학생이 AI에게 가르친다" 메커니즘에, RPG의 **캐릭터 육성 루프**를 결합한 구조다.
단순 게임화(뱃지·포인트)가 아니라 **성장 서사(Growth Narrative)** 가 있어 장기 리텐션에 훨씬 유리하다.
Duolingo의 스트릭, 포켓몬 육성, 다마고치가 왜 중독성을 갖는지를 교육에 정확히 이식했다.

#### 2. 망각 곡선 설계가 핵심 차별화 포인트
이 아이디어에서 가장 독창적인 부분은 **페르소나가 잊어버린다**는 점이다.
페르소나가 완벽하게 기억한다면, 사용자는 한 번 가르치고 영원히 레벨업만 하면 된다. 이는 반복 복습이 일어나지 않는다는 치명적 결함으로 이어진다.
그러나 에빙하우스 망각 곡선을 페르소나에 적용하면, 사용자는 **페르소나의 기억이 흐려지기 전에 다시 가르쳐야 한다는 내적 동기**가 생긴다. 이는 간격 반복(Spaced Repetition)을 게임 속에 자연스럽게 내재화한 설계다.

#### 3. 감정적 투자(Emotional Investment)의 극대화
"내 학생"이 시험에서 낮은 점수를 받으면, 사용자는 **자신이 잘 못 가르쳤다는 책임감**을 느낀다.
이 감정은 일반적인 "내 점수"보다 훨씬 강력한 동기 부여 요소가 된다.
Tamagotchi, 닌텐독스, 스타듀밸리에서 검증된 "돌봄 책임감" 심리를 학습에 연결한 점이 뛰어나다.

#### 4. "함께 시험 치르기" 메커니즘의 독창성
학생과 교사가 **같은 시험지를 동시에 풀고 합산 점수로 레벨업이 결정**된다는 구조는
현재 국내외 에듀테크 시장 어디에도 존재하지 않는다. 이 단 하나의 메커니즘만으로도 창의성 최고점을 받을 수 있다.

### 비판적 검토 및 우려 사항 ⚠️

#### 비판 1: 합산 점수 구조의 논리 허점
**문제**: 사용자가 100점을 받아도 페르소나가 0점이면 레벨업 실패. 반대로 페르소나가 우연히 고점이면 사용자가 낮은 점수를 받아도 통과할 수 있다.

**결과**: 사용자가 잘 가르치는 것보다 "페르소나를 잘 조작"하는 방향으로 최적화할 수 있다. 목표 이탈.

**보완안**: 합산 점수와 함께 **개인 최저 점수 조건** 병행 적용.
- 예: 합산 150점 이상 + 사용자 최소 60점 + 페르소나 최소 40점
- 비율: 사용자 점수 60% + 페르소나 점수 40% 가중 합산

---

#### 비판 2: "초등학교 1학년 수준 시험지" 생성의 커리큘럼 정합성 문제
**문제**: LLM이 임의로 "초1 수준"의 시험을 생성할 경우, 실제 2022 개정 교육과정과 괴리가 생긴다.
실제 초1 수학은 한 자리 덧셈·뺄셈에 한정되지만, LLM은 자의적으로 더 쉽거나 어려운 문제를 낼 수 있다.

**결과**: 사용자가 "이게 진짜 초1 수준이 맞나?" 하는 신뢰도 문제가 발생한다.

**보완안**: 각 학년·과목별 **출제 범위 스펙 테이블**을 프롬프트에 명시적으로 주입.
```
예) 초등학교 1학년 수학 출제 범위:
- 9까지의 수, 한 자리 수 덧셈·뺄셈
- 시계 보기 (정각, 30분)
- 기본 도형 (삼각형, 사각형, 원)
```
이 스펙 테이블을 MVP에서는 수학·국어 2과목 × 초1~중3(9학년) 수준으로 제작. 나머지 과목과 고등·대학은 이후 확장.

---

#### 비판 3: 망각 곡선 파라미터 설정의 밸런싱 난이도
**문제**: 망각 속도가 너무 빠르면 → 사용자가 매일 재교육에 지쳐 이탈.
너무 느리면 → 망각 기능이 무의미해져 게임성 소멸.

**결과**: 잘못된 파라미터가 핵심 재미 요소를 망가뜨린다.

**보완안**: 에빙하우스 망각 곡선을 직접 적용하되, **강의 횟수에 따라 망각 속도를 늦추는 보너스** 적용.
```
기본 망각 공식:
  기억률(t) = e^(-t / S)
  t: 마지막 학습 이후 경과 일수
  S: 안정성 계수 (최초 학습 시 S=1, 복습 1회 S=2, 복습 2회 S=4, ...)

MVP 단순화 버전:
  - 마지막 수업 후 1일: 기억률 80%
  - 3일: 기억률 60%
  - 7일: 기억률 35%
  - 14일: 기억률 15%
  - 복습할 때마다 기억률 리셋 + 다음 감소 속도 20% 느려짐
```

---

#### 비판 4: 1주일 개발 범위 초과 위험
**문제**: 원래 MVP였던 TeachBack AI(채팅 + 점수)에 추가된 기능:
- 페르소나 시스템 (상태, 레벨, 기억 데이터)
- 망각 곡선 엔진
- 시험 생성 + 동시 응시 + 합산 채점
- 레벨업 판정 시스템
- 성장 히스토리 UI

이 모든 것을 7일에 완성하면 각 기능의 완성도가 낮아질 수 있다.

**보완안**: MVP 핵심 루프를 3단계로 엄격히 제한하고 나머지는 "시연용 Mock 데이터"로 대체.
→ 자세한 범위는 PART 3에서 확정.

---

#### 비판 5: "페르소나가 시험을 친다"는 UX 전달의 어려움
**문제**: 페르소나가 시험을 치르는 과정이 사용자에게 시각적으로 명확하지 않으면,
"AI가 그냥 랜덤으로 답 냈겠지"라고 느껴져 감정적 투자가 깨진다.

**보완안**: 페르소나의 시험 응답에 **생각하는 과정 텍스트를 함께 표시**.
```
[내 제자 민이의 답안]
"음... 선생님이 이차방정식 풀이 방법 알려줬는데,
 근의 공식은 기억나는데 판별식 D가 뭔지 좀 헷갈려..."
(답: 해 없음 → 오답)
```
이 "생각하는 텍스트"가 페르소나를 진짜 사람처럼 느끼게 만드는 핵심 UX 장치다.

## PART 2 — 보완 아이디어 추가 제안

### 보완 1: 페르소나 개성(Personality) 시스템
모든 학생이 같은 방식으로 배우지 않는다. 페르소나에 **학습 스타일 개성**을 부여하면 애착이 깊어진다.

| 개성 유형 | 특징 | 게임적 반영 |
|---------|------|----------|
| 호기심형 | 질문을 많이 함 | 가르칠 때 AI가 더 많은 질문을 던짐 → 사용자가 더 깊이 설명해야 함 |
| 신중형 | 천천히 배우지만 오래 기억 | 망각 곡선 감소 속도 20% 느림 → 복습 부담 적음 |
| 덤벙이형 | 빠르게 배우지만 잘 잊음 | 망각 곡선 감소 속도 30% 빠름 → 자주 복습 필요 |
| 완벽주의형 | 이해가 완전해야 다음으로 | 시험 난이도가 10% 높아짐, 대신 레벨업 후 보너스 점수 |

MVP에서는 **온보딩 시 4가지 중 1가지 선택**으로 구현. 이후 AI가 대화 패턴으로 자동 감지하는 방식으로 고도화.

---

### 보완 2: "가르치기 품질" 피드백 루프
현재 구조에서는 사용자가 대충 가르쳐도 페르소나가 학습한다. 이렇게 되면 사용자의 설명 능력 향상이라는 핵심 가치가 희석된다.

**제안**: 각 수업 세션 후 **"이 수업의 교육 품질 점수"** 를 사용자에게 피드백.
```
[수업 품질 리포트]
오늘 이차방정식 수업 품질: 72점
- 잘한 점: 근의 공식 유도 과정을 단계적으로 설명했어요 ✅
- 부족한 점: 판별식 개념을 설명 안 하고 넘어갔어요 ⚠️
- 민이의 이해도: 이 수업 이후 판별식 관련 문제 정답률 예측 45%
```
이 피드백이 사용자 자신의 메타인지 향상 도구가 된다 → 원래 TeachBack AI의 핵심 목표 유지.

---

### 보완 3: 레벨업 졸업식 연출
레벨업 순간은 **감정적 클라이맥스**여야 한다. 단순한 숫자 변경으로 처리하면 게임성이 반감된다.

**제안**: 레벨업 시 짧은 **졸업 연출 화면**:
```
[초등학교 1학년 졸업!]
"선생님, 저 이제 2학년 됐어요! 덧셈 뺄셈은 이제 자신 있어요.
 2학년에서는 곱셈 배우고 싶어요!"
```
페르소나가 직접 소감을 말하는 형식. 이 한 줄 텍스트가 사용자의 감정적 성취감을 극대화한다.

---

### 보완 4: "내 제자의 약점 노트"
페르소나가 시험에서 반복적으로 틀리는 부분을 **자동 약점 노트**로 시각화.

```
[민이의 약점 노트]
수학: 판별식 (3번 연속 오답), 인수분해 (2번 오답)
국어: 피동/사동 구분 (4번 연속 오답)
```
이것이 역설적으로 **사용자 자신의 취약점 진단**이기도 하다. 잘 못 가르쳤다 = 사용자도 잘 모른다는 반증이기 때문.

## PART 3 — 최종 통합 기획안 확정

### 솔루션명: **"나의 제자 (My Jeja)"**
> *"가르치면서 내가 더 배운다. 내 제자가 성장하면, 나도 성장한다."*

```
핵심 루프:

  [오늘 배운 것 가르치기]
        ↓
  페르소나가 학습 (망각 곡선 시작)
        ↓
  [시험 요청] → 페르소나 레벨 맞춤 시험 생성
        ↓
  사용자 + 페르소나 동시 응시
        ↓
  합산 점수 판정 → 통과 시 레벨업 / 실패 시 재교육
        ↓
  [약점 노트 업데이트] → 다시 가르치기로
```

---

### 사용자 경험 흐름 (User Flow)

```
온보딩
  └─ 제자 이름 짓기 + 개성 선택 (호기심형/신중형/덤벙이형/완벽주의형)
  └─ 시작 레벨: 초등학교 1학년, 과목 선택 (수학 or 국어 택 1 — MVP)

[홈 화면]
  ├─ 내 제자 상태 카드
  │    ├─ 현재 레벨 (초1 ~ 대학원)
  │    ├─ 기억 유지율 (오늘 기준)
  │    └─ 다음 레벨업까지 남은 것
  ├─ [가르치기] 버튼
  └─ [시험 치르기] 버튼

[가르치기 세션]
  ├─ 오늘 가르칠 개념 입력
  ├─ 채팅 UI: 사용자가 설명 → 페르소나가 소크라테스 질문
  ├─ 페르소나 개성에 따라 질문 스타일 다름
  └─ 세션 종료: 수업 품질 점수 + 페르소나 기억 업데이트

[시험 치르기]
  ├─ 페르소나 현재 레벨 + 약점 반영한 시험지 자동 생성 (5문제)
  ├─ 사용자 풀기 (제한 시간 10분)
  ├─ 페르소나 풀기 (기억 상태 기반, 생각 과정 텍스트 함께 표시)
  ├─ 채점: 사용자 60% + 페르소나 40% 합산
  ├─ 통과 (합산 75점↑ + 사용자 최소 50점 + 페르소나 최소 30점)
  │    └─ 레벨업 졸업 연출
  └─ 미통과: 약점 노트 업데이트 + 재교육 가이드

[성장 기록]
  ├─ 레벨별 졸업 날짜 타임라인
  ├─ 과목별 이해도 변화 그래프
  └─ 누적 약점 태그 히트맵
```

### MVP 범위 확정 (7일 기준 엄격 제한)

#### ✅ 포함 (필수 핵심 루프)

| 기능 | 구현 방식 | 예상 개발 시간 |
|------|---------|:----------:|
| 온보딩 (이름 + 개성 선택) | 정적 화면 4개 | 4h |
| 가르치기 채팅 UI | React-Native + FastAPI 스트리밍 | 12h |
| 소크라테스 AI 응답 | Claude API 프롬프트 | 6h |
| 수업 품질 점수 산출 | Claude API (세션 종료 후 평가) | 4h |
| 망각 곡선 엔진 | 단순 시간 기반 decay 함수 (DB에 저장) | 6h |
| 시험 생성 (수학 초1~중3 범위) | Claude API + 학년별 출제 범위 스펙 주입 | 8h |
| 사용자 시험 응시 UI | 5지선다 + 주관식 혼합 | 8h |
| 페르소나 시험 응시 + 생각 텍스트 | Claude API (기억 상태 컨텍스트 주입) | 8h |
| 합산 채점 + 레벨업 판정 | 서버 로직 | 4h |
| 레벨업 졸업 연출 화면 | 단순 애니메이션 화면 | 4h |

### 핵심 프롬프트 설계 확정 (Prompt Engineering 지침서)

#### Prompt 1: 소크라테스 교사 (가르치기 세션)

```
SYSTEM:
너는 [레벨: 초등학교 {n}학년] 수준의 학생 페르소나야.
이름은 '{name}'이고, 학습 성격은 '{personality}'야.

성격별 행동 지침:
- curious(호기심형): 설명 중 "왜요?", "그럼 이건요?" 질문 2~3회 이상
- careful(신중형): "잠깐, 앞에 말씀하신 부분을 한 번 더 정리해줄 수 있나요?"
- clumsy(덤벙이형): 가끔 엉뚱한 방향으로 이해 → "아, 그럼 이렇게 되는 건가요? (틀린 해석)"
- perfectionist(완벽주의형): "그 부분은 이해했는데, 예외 케이스는요?"

절대 규칙:
1. 너는 배우는 입장이다. 답을 먼저 말하지 않는다.
2. 선생님(사용자)이 설명한 내용에서 논리 빈틈을 발견하면 그것을 질문으로 찌른다.
3. 대화 중 1번, 의도적으로 약간 틀린 이해를 표현해 선생님이 교정하도록 유도한다.
4. 현재 기억하고 있는 개념 목록: {memory_context} (이것만 알고 있다고 전제)

USER: [사용자의 설명 텍스트]
```

#### Prompt 2: 시험 생성기

```
SYSTEM:
한국 교육과정 기준 {grade}학년 수학 시험 문제 5개를 생성해라.

출제 범위 스펙:
{grade_spec}  ← DB에서 학년별 스펙 테이블 주입

조건:
- 5지선다 3문제 + 단답형 2문제
- 난이도: 하(1) × 2, 중(2) × 2, 상(3) × 1
- 페르소나의 누적 약점 태그: {weak_tags} → 이 개념에서 최소 2문제 출제
- 출력 형식: JSON
  { questions: [ { id, type, content, options?, answer, concept_tag } ] }
```

#### Prompt 3: 페르소나 시험 응시

```
SYSTEM:
너는 {grade}학년 수준의 학생 '{name}'이다.
현재 기억하고 있는 개념과 기억률:
{memory_with_retention}
(기억률이 낮은 개념은 불완전하게 기억하거나 틀릴 수 있다)

각 문제에 대해:
1. 짧은 "생각 텍스트" (1~2줄, 학생이 고민하는 것처럼)
2. 최종 답변

기억률 < 40%인 개념이 포함된 문제는 70% 확률로 오답 처리.
기억률 40~70%는 40% 확률로 오답.
기억률 > 70%는 10% 확률로 오답.

출력 형식: JSON
{ answers: [ { question_id, thought, answer } ] }
```

#### Prompt 4: 수업 품질 평가자

```
SYSTEM:
다음 수업 대화 내용을 분석하여 수업 품질을 평가하라.

대화 내용: {session_messages}
가르치려 한 개념: {concept}
학년 수준: {grade}

평가 기준 (각 25점):
1. 정확성: 설명에 오류가 없는가?
2. 깊이: 표면적 암기가 아닌 원리 설명이 있는가?
3. 예시: 구체적 예시를 들었는가?
4. 완결성: 핵심 내용 누락 없이 다뤘는가?

출력 형식: JSON
{
  "score": 0~100,
  "good_points": ["..."],
  "weak_points": ["..."],  ← persona_memory에 저장될 약점
  "predicted_retention": 0.0~1.0
}
```

## AI 협업 리포트 메모 (기획 3단계)

**이 문서에서의 AI 역할**:
사용자가 제안한 아이디어의 강점 4가지와 잠재 위험 5가지를 구조적으로 분석하고,
각 위험에 대한 구체적 보완안을 교육학·게임 심리학·기술 구현 가능성의 세 축으로 검토했다.

**핵심 설계 결정**:
- 합산 점수 + 개인 최저 조건 병행 (게임 밸런싱)
- 망각 곡선 단순화 (에빙하우스 기반이지만 MVP에서는 단계적 decay 함수로 근사)
- 페르소나 "생각 텍스트" UX (감정적 투자 유지 장치)
- MVP 수학 1과목 × 초1~중3 9레벨로 범위 고정 (1주 완성 보장)

**참조 이론**:
- Ebbinghaus Forgetting Curve (1885)
- Protégé Effect (Palinscar & Brown, 1984)
- Feynman Technique
- ARCS 동기 모형 (Keller, 1987) — 주의(Attention)·관련성(Relevance)·자신감(Confidence)·만족감(Satisfaction)
