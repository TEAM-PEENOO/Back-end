# AI 학생 학습 메커니즘 — 핵심 구현 로직

> AI 학생이 "학습하고 기억한다"는 것의 실제 구현 방식

---

## 핵심 전제: AI 학생의 "학습"은 실제 ML이 아니다

많은 사람이 착각하는 부분이지만, AI 학생이 "배운다"는 것은
**Claude를 파인튜닝하거나 모델을 업데이트하는 것이 아니다.**

```
실제 구현 =
  DB에 "무엇을 가르쳤는가" 저장
  + 수학 공식으로 "얼마나 기억하는가" 계산
  + 프롬프트에 그 정보 주입
  + Claude가 해당 기억 상태로 연기(simulate)
```

LLM은 원래 **stateless**하다.
"기억하는 척"은 전부 **컨텍스트 주입(Context Injection)** 으로 구현된다.

---

## 구현 3단계

### 1단계 — 수업 후 "기억 요약" 저장

가르치기 세션이 종료되면, Claude에게 수업 내용을 JSON으로 요약하도록 요청한다.
전체 대화 메시지를 그대로 저장하면 이후 토큰이 폭발하므로,
**세션 종료 시마다 요약본을 생성하고 원본 메시지는 버린다.**

**요약 요청 프롬프트:**
```
이 수업 대화에서 학생이 배운 핵심 개념을 JSON으로 요약해줘.
각 개념은 이름과 한 줄 요약으로 구성한다.

출력 형식:
{
  "concepts": [
    { "name": "개념명", "summary": "핵심 내용 한 줄 요약" }
  ]
}
```

**저장 결과 예시:**
```json
{
  "concepts": [
    {
      "name": "CSS 박스 모델",
      "summary": "content/padding/border/margin 4층 구조. 배경색은 padding까지 적용됨."
    },
    {
      "name": "box-sizing",
      "summary": "border-box로 설정하면 padding/border가 width에 포함되어 레이아웃 계산이 편해짐."
    }
  ]
}
```

→ 이 요약이 `persona_memory` 테이블의 `concept`과 `summary` 컬럼에 저장된다.

---

### 2단계 — 기억률은 서버에서 수학 계산

기억률은 DB에 저장된 값이 아니라, **API 요청 시마다 서버에서 실시간 계산**한다.

**에빙하우스 망각 곡선 공식:**
```
기억률(t) = e^(-t / S)

t: 마지막 수업 이후 경과 일수
S: 안정성 계수 (복습할수록 증가)
```

**Python 구현:**
```python
import math
from datetime import datetime

def get_retention(last_taught_at: datetime, stability: float) -> float:
    """
    현재 기억률 계산 (0.0 ~ 1.0)
    last_taught_at: 마지막으로 가르친 시각
    stability: 안정성 계수 (최초=1, 복습 1회=2, 복습 2회=4, 3회+=8)
    """
    t = (datetime.now() - last_taught_at).days
    return math.exp(-t / stability)


def get_stability(taught_count: int) -> float:
    """복습 횟수에 따른 안정성 계수"""
    if taught_count <= 1:
        return 1.0
    elif taught_count == 2:
        return 2.0
    elif taught_count == 3:
        return 4.0
    else:
        return 8.0
```

**기억률 단계별 게임 표시:**

| 기억률 | 게임 표시 | 색상 |
|:------:|:--------:|:----:|
| 80%+ | 선명 | 🟢 |
| 60~80% | 흐릿해지는 중 | 🟡 |
| 40~60% | 많이 흐릿함 | 🟠 |
| 20~40% | 거의 잊어버림 | 🔴 |
| ~20% | 잊어버림 | ⚫ |

---

### 3단계 — 시험 시 기억 상태를 프롬프트에 주입

시험을 생성할 때는 사용자가 가르친 요약본들을 묶어서 넘기고,
AI 학생이 시험을 풀 때는 현재 기억 상태(기억률)를 주입한다.

**AI 학생 시험 응시 프롬프트:**
```
SYSTEM:
너는 제이라는 학생이다.
현재 기억 상태 (방금 서버에서 계산된 값):

- CSS 박스 모델:  기억률 78% → 비교적 잘 기억함
- box-sizing:    기억률 23% → 거의 잊어버림
- Flexbox:       기억률  5% → 완전히 잊음

오답 확률 규칙:
  기억률 < 30%  → 80% 확률로 오답
  기억률 30~60% → 45% 확률로 오답
  기억률 > 60%  → 10% 확률로 오답

각 문제에 대해:
1. 짧은 "생각 텍스트" (1~2줄, 학생이 고민하는 것처럼 자연스럽게)
2. 최종 답변

출력 형식: JSON
{ "answers": [ { "question_id": "...", "thought": "...", "answer": "..." } ] }
```

## 시험 생성 로직 — 가르친 내용 기반

외부 교과서나 정규 커리큘럼 스펙이 아닌,
**사용자가 실제로 가르친 내용(요약본)만을 기반으로 문제를 생성**한다.

**시험 생성 흐름:**
```
1. 현재 단계(Stage)에 포함된 커리큘럼 항목 목록 조회
2. 해당 항목들의 teaching_sessions → 요약본 묶음 조회
3. weak_point_tags (누적 오답 개념) 조회
4. Claude에게 넘겨 문제 5개 생성
```

**시험 생성 프롬프트:**
```
아래는 학생이 선생님에게 직접 배운 수업 요약이다.
이 내용만을 바탕으로 시험 문제 5개를 만들어라.
교과서나 외부 지식을 사용하지 않는다.
오직 아래 수업 요약에서 설명된 내용만 출제한다.

[수업 요약]
{teaching_summaries}

[누적 약점 개념 — 우선 출제]
{weak_tags}

출제 조건:
- 5지선다 3문제 + 단답형 2문제
- 난이도: 하 × 2, 중 × 2, 상 × 1
- 약점 개념에서 최소 2문제 출제

출력 형식: JSON
{
  "questions": [
    {
      "id": "q1",
      "type": "multiple_choice" | "short_answer",
      "content": "문제 내용",
      "options": ["①", "②", "③", "④", "⑤"],  // 단답형은 null
      "answer": "정답",
      "concept_tag": "관련 개념명",
      "difficulty": 1 | 2 | 3
    }
  ]
}
```

## 토큰 폭발 방지 전략

수업이 쌓일수록 전체 대화를 컨텍스트에 넣으면 토큰이 초과된다.

**해결책: 세션 종료마다 요약 → 요약본만 보존**

```
[수업 세션 종료 시 처리 흐름]

1. 전체 대화 메시지 (messages JSONB) → Claude에게 요약 요청
2. 요약 결과를 persona_memory에 저장
3. teaching_sessions.messages는 빈 배열로 초기화 (또는 별도 archive)

[시험 생성 시 사용하는 데이터]
→ persona_memory의 summary 필드들만 묶어서 전달
→ 수백 번 수업을 해도 토큰 양이 선형적으로 증가하지 않음
```

## 실제로 복잡한 부분

AI 학생 구현 자체보다 **아래 부분이 실제 개발 난이도가 높다:**

| 항목 | 난이도 | 이유 |
|------|:------:|------|
| 커리큘럼 + 단계 설정 UI | ★★★ | 항목 순서 관리, 단계 묶기 인터랙션 |
| 가르치기 채팅 스트리밍 | ★★★ | FastAPI SSE + React-Native 연결 |
| 합산 채점 + 진급 로직 | ★★ | 조건 분기 처리 |
| 망각 기억률 계산 | ★ | 수학 함수 하나로 끝 |
| AI 학생 "학습" 구현 | ★ | DB 저장 + 프롬프트 주입으로 끝 |

## 기간이 촉박할 때 버려도 되는 것

| 기능 | 대안 |
|------|------|
| 드래그 앤 드롭 커리큘럼 순서 변경 | 위/아래 버튼으로 대체 |
| 단계 설정 전용 화면 | 항목마다 단계 드롭다운 선택으로 단순화 |
| 기억률 애니메이션 | 숫자 % + 색상 텍스트로 대체 |
| 망각 푸시 알림 | 홈 화면 진입 시 배지 표시로 대체 |
| 개성 자동 추천 (응답 패턴 분석) | 온보딩에서 직접 선택만 제공 |

## 전체 데이터 흐름 요약

```
[가르치기]
  사용자 설명 입력
    → FastAPI: Claude API 호출 (소크라테스 프롬프트 + memory_context 주입)
    → Claude: 소크라테스 질문 반환
    → 세션 종료 시: 요약 요청 → persona_memory 업데이트 (stability 증가)

[홈 화면 로딩]
  → DB에서 persona_memory 조회
  → 각 concept마다 get_retention() 계산
  → 기억률 낮은 항목 강조 표시

[단계 시험]
  사용자가 "시험 요청" 버튼 클릭
    → 현재 단계 curriculum_items의 teaching 요약본 묶음 조회
    → Claude: 시험지 5문제 생성
    → 사용자 응시 (UI)
    → 각 concept 기억률 계산 → Claude: AI 학생 응시 (오답 확률 적용)
    → 합산 채점 → 진급 판정
    → 통과 시: stages.passed = true, current_stage_id 업데이트
    → 실패 시: weak_point_tags 업데이트
```
