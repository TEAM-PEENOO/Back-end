## 1. Executive Summary

**나의 제자(My Jeja)** 는 "AI가 학생을 가르친다"는 기존 에듀테크의 방향을 완전히 뒤집은 모바일 학습 앱입니다.

사용자는 자신이 오늘 배운 지식을 AI 학생 페르소나에게 직접 가르칩니다. AI는 소크라테스 방식의 질문으로 응답하고, 사용자의 설명 속 오개념과 논리 공백을 스스로 발견하도록 유도합니다.

이 과정에서 AI 학생은 실제 인간처럼 에빙하우스 망각 곡선을 따라 배운 내용을 잊어갑니다. 사용자와 AI 학생은 주기적으로 **함께 시험**을 치르고, 두 점수의 합산이 기준을 넘으면 AI 학생이 다음 학년으로 승급합니다.

처음 시작 시 **배치고사**를 통해 사용자의 실제 수준을 측정하고, AI 학생의 시작 레벨을 자동으로 설정합니다. AI 학생은 **4가지 학습 개성** 중 하나를 가지며, 이 개성이 질문 방식·망각 속도·시험 난이도에 영향을 줍니다.

> **핵심 가치 한 줄 요약**:
> *"가르치면서 내가 더 배운다. 내 제자가 성장하면, 나도 성장한다."*

### 학생이 겪는 구조적 문제

| # | 페인포인트 | 현황 |

| ② | **수동적 학습 구조**: 모든 에듀테크가 AI→학생 방향 | 듣기/읽기 학습 효율 5~10%, 가르치기 효율 90% |

```
공통 문제: 모두 "AI가 교사, 사람이 학생" 구도

```

## 3. 솔루션 개요 — 나의 제자 (My Jeja)

### 핵심 아이디어: 교육의 방향을 역전한다

```
기존 모든 AI 교육앱:   AI  ──[설명]──▶  학생    (수동적)
나의 제자:            학생 ──[설명]──▶  AI 학생  (능동적)
```

### 교육학적 근거 3가지

| 이론 | 내용 | 앱에서의 적용 |
|------|------|------------|
| **파인만 기법** (Feynman Technique) | 개념을 쉽게 설명할 수 있어야 진짜 이해한 것 | 사용자가 AI에게 개념을 설명하는 전체 구조 |
| **프로테제 효과** (Protégé Effect) | 누군가를 가르친다는 생각이 학습 동기·깊이 모두 향상 | AI 학생 페르소나에 대한 감정적 책임감 |
| **에빙하우스 망각 곡선** (Ebbinghaus, 1885) | 복습 없이는 시간이 지날수록 기억 감소 | 페르소나 기억 상태에 망각 곡선 적용 |

### 게임화 핵심 루프

```
┌─────────────────────────────────────────────────────┐
│                   나의 제자 핵심 루프                    │
│                                                      │
│  [배치고사] → 내 수준 측정 → AI 학생 시작 레벨 자동 설정    │
│       ↓                                              │
│  [오늘 배운 것 가르치기]                                 │
│       ↓                                              │
│  AI 학생이 소크라테스 질문 → 사용자 오개념 스스로 발견       │
│       ↓                                              │
│  수업 품질 점수 + 페르소나 기억 업데이트 (망각 곡선 시작)     │
│       ↓                                              │
│  [시험 치르기 요청]                                     │
│       ↓                                              │
│  페르소나 레벨 맞춤 시험 자동 생성                         │
│       ↓                                              │
│  사용자 응시 ←──── 동시 ────→ 페르소나 응시 (망각 반영)      │
│       ↓                                              │
│  합산 점수 판정 (사용자 60% + 페르소나 40%)                │
│       ↓                          ↓                   │
│  ✅ 통과 → 레벨업 졸업 연출    ❌ 실패 → 약점 재교육          │
└─────────────────────────────────────────────────────┘
```

## 4. 핵심 메커니즘

### 4-1. 망각 곡선 엔진

AI 학생은 단순히 배운 내용을 무제한 기억하지 않습니다. 에빙하우스 망각 곡선을 따라 시간이 지날수록 기억이 흐려집니다.

**기억률 계산 공식**:

```
기억률(t) = e^(-t / S)

t: 마지막 수업 이후 경과 일수
S: 안정성 계수 (복습할수록 증가)
   - 최초 학습:    S = 1  → 7일 후 기억률 약 37%
   - 1회 복습:     S = 2  → 14일 후 기억률 약 37%
   - 2회 복습:     S = 4  → 28일 후 기억률 약 37%
   - 3회 이상:     S = 8  → 56일 후 기억률 약 37%
```

**MVP 단순화 버전 (시간 기반 단계적 감소)**:

| 마지막 수업 후 경과 | 기억률 | 게임적 표시 |
|:---------------:|:-----:|:---------:|
| 0~1일 | 95% | 🟢 선명 |
| 2~3일 | 75% | 🟡 흐릿해지는 중 |
| 4~7일 | 50% | 🟠 많이 흐릿함 |
| 8~14일 | 25% | 🔴 거의 잊어버림 |
| 15일+ | 10% | ⚫ 잊어버림 |

**망각과 오답 확률 연결**:

| 기억률 | 시험 오답 확률 |
|:------:|:-----------:|
| 80%+ | 5% |
| 60~80% | 20% |
| 40~60% | 45% |
| 20~40% | 70% |
| ~20% | 90% |

### 4-2. 합산 채점 시스템

```
최종 점수 = (사용자 점수 × 0.6) + (페르소나 점수 × 0.4)

레벨업 조건:
  ✅ 최종 점수 ≥ 75점
  ✅ 사용자 점수 ≥ 50점 (개인 최저 조건)
  ✅ 페르소나 점수 ≥ 30점 (개인 최저 조건)
  → 세 조건 모두 충족 시 레벨업

실패 시: 약점 노트 업데이트 + 재교육 가이드 제공
```

## 5. 배치고사 시스템 (신규 기능)

### 5-1. 개요

앱을 처음 시작할 때, 사용자는 "처음부터 시작(초1)"과 "내 수준 측정하기(배치고사)" 중 선택할 수 있습니다.

배치고사는 **적응형 컴퓨터 테스트(CAT: Computerized Adaptive Testing)** 방식으로, 약 10~12문제 안에 사용자의 실제 수준을 정확히 파악합니다.

### 5-2. 적응형 알고리즘

```
시작 레벨: 4 (초등학교 4학년 — 전체 9단계의 중간)

규칙:
  문제 정답 → 다음 문제 난이도 상향 (+1 레벨)
  문제 오답 → 다음 문제 난이도 하향 (-1 레벨)
  동일 레벨 2회 연속 정답 → 해당 레벨 확정 및 종료
  동일 레벨 2회 연속 오답 → 하위 레벨 확정 및 종료
  12문제 초과 → 현재 레벨 강제 확정

결과 예시:
  초4 정답 → 초5 정답 → 초6 오답 → 초5 정답 → 초5 확정 → 종료
  (7문제로 수렴)
```

### 5-3. 배치고사 흐름

```
[배치고사 시작]
  └─ 안내 화면: "약 10문제로 내 수준을 파악해요 (5~7분 소요)"

[문제 응시]
  └─ 한 번에 1문제씩 표시
  └─ 5지선다, 제한 시간 없음
  └─ 이전 문제로 돌아갈 수 없음

[결과 화면]
  ├─ "민이의 시작 레벨: 초등학교 5학년으로 결정되었어요!"
  ├─ 강점 개념: "분수 계산, 기본 도형은 이미 잘 알고 있어요"
  ├─ 약점 개념: "소수의 나눗셈이 조금 약해요 — 함께 보완해봐요"
  └─ [제자와 여정 시작하기] 버튼
```

### 5-4. 배치고사 결과의 활용

| 결과 항목 | 활용 방식 |
|---------|---------|
| 확정된 시작 레벨 | `personas.current_level` 초기값 설정 |
| 정답 처리된 하위 개념들 | `persona_memory`에 기억률 0.90으로 등록 |
| 오답 처리된 개념들 | `weak_point_tags`에 초기 등록 |
| 현재 레벨 정답 개념들 | `persona_memory`에 기억률 0.75로 등록 |

## 6. 학습자 개성 시스템

### 6-1. 개요

AI 학생 페르소나는 4가지 학습 개성 중 하나를 가집니다.
개성은 **온보딩 시 사용자가 직접 선택**하거나,
**배치고사 응답 패턴(응답 속도, 오답 유형)을 AI가 분석해 자동 추천**합니다.

개성은 단순한 스타일 차이가 아니라, **망각 속도·질문 빈도·시험 난이도·레벨업 조건**에 실질적으로 영향을 주는 게임 파라미터입니다.

### 6-2. 4가지 개성 상세

#### 🔵 호기심쟁이 (Curious)

```
성격: 배우는 것을 즐기고, 끊임없이 더 알고 싶어한다
말투: 밝고 에너지 넘침. "왜요?", "그럼 이건요?", "더 알려주세요!"

게임 파라미터:
  - 망각 속도:      표준 (×1.0)
  - 질문 빈도:      높음 (세션당 3~4회 질문)
  - 시험 난이도:    표준
  - 레벨업 조건:    합산 75점 이상

전략적 특성:
  가르치는 세션이 길어지는 대신, 사용자가 개념을 더 깊이 파야 한다.
  다방면으로 질문을 던져 설명의 빈틈을 자연스럽게 드러낸다.
```

#### 🟢 신중이 (Careful)

```
성격: 꼼꼼하고 천천히 배우지만, 한번 이해하면 잘 안 잊는다
말투: 조용하고 정중함. "제가 이해한 게 맞는지 확인해도 될까요?"

게임 파라미터:
  - 망각 속도:      느림 (×0.7 — 같은 기간 동안 덜 잊음)
  - 질문 빈도:      낮음 (세션당 1~2회, 정확한 포인트만)
  - 시험 난이도:    표준
  - 레벨업 조건:    합산 70점 이상 (약간 쉬운 조건)

전략적 특성:
  복습 부담이 적은 대신, 가르치는 세션에서 충분히 이해시키지 않으면
  취약점이 오래 남는다. 수업 품질이 중요한 개성.
```

#### 🟡 덤벙이 (Clumsy)

```
성격: 빠르게 배우지만 금방 잊어버리고, 가끔 엉뚱한 방향으로 이해한다
말투: 활발하지만 급함. "아 알겠어요! 그럼 이렇게 되는 거죠? (틀린 해석)"

게임 파라미터:
  - 망각 속도:      빠름 (×1.5 — 더 빨리 잊음)
  - 질문 빈도:      보통 (2~3회, 하지만 엉뚱한 질문 포함)
  - 시험 난이도:    표준
  - 레벨업 조건:    합산 80점 이상 (약간 어려운 조건)

전략적 특성:
  가르치는 것 자체는 재미있지만, 복습을 자주 해야 한다.
  덤벙이를 선택한 사용자는 짧고 자주 가르치는 패턴이 유리하다.
```

#### 🔴 완벽주의자 (Perfectionist)

```
성격: 기본 개념을 빠르게 습득하지만, 반드시 심화까지 알아야 직성이 풀린다
말투: 날카롭고 분석적. "그 개념의 예외 케이스는요?", "실제로 어떻게 쓰이나요?"

게임 파라미터:
  - 망각 속도:      느림 (×0.8)
  - 질문 빈도:      높음 (3~4회, 하지만 심화 질문 위주)
  - 시험 난이도:    +10% 어려움 (상위 난이도 문제 비율 증가)
  - 레벨업 조건:    합산 85점 이상 (가장 어려운 조건)
  - 레벨업 보너스:  달성 시 다음 레벨 시작 기억률 +10%

전략적 특성:
  가장 도전적인 개성. 통과 기준이 높은 대신, 레벨업 보너스가 있다.
  사용자 자신도 심화 개념까지 완벽히 이해해야 통과할 수 있다.
```

### 6-3. 개성 선택 방식 2가지

**방식 A: 온보딩 직접 선택**

```
[어떤 제자를 키울까요?]

🔵 호기심쟁이  — 궁금한 게 많아요, 질문 폭발 주의!
🟢 신중이     — 천천히 배우지만 오래 기억해요
🟡 덤벙이     — 빠르지만 자주 잊어요, 복습이 중요!
🔴 완벽주의자 — 심화까지 파야 직성이 풀려요, 도전적!
```

**방식 B: 배치고사 패턴 자동 분석 (AI 추천)**

```
배치고사 응답 데이터에서:
- 평균 응답 속도가 빠르고 오답률이 높음    → 덤벙이 추천
- 평균 응답 속도가 느리고 오답률이 낮음    → 신중이 추천
- 오답 유형이 "심화 개념"에 집중           → 완벽주의자 추천
- 전반적으로 고른 오답 분포               → 호기심쟁이 추천

"배치고사 결과를 보니 민이는 [덤벙이] 스타일 같아요!
 빠르게 배우지만 복습이 중요한 제자예요. 어때요?"
[이대로 시작] / [직접 선택할게요]
```

## 10. 데이터베이스 스키마

```sql
-- 사용자 계정
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  created_at  TIMESTAMP DEFAULT NOW()
);

-- AI 학생 페르소나 (MVP: 사용자당 1개)
CREATE TABLE personas (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID REFERENCES users(id) ON DELETE CASCADE,
  name          TEXT NOT NULL,
  personality   TEXT NOT NULL CHECK (personality IN ('curious','careful','clumsy','perfectionist')),
  current_level INT  NOT NULL DEFAULT 1,  -- 1=초1, 2=초2, ..., 9=중3
  subject       TEXT NOT NULL DEFAULT 'math',
  placement_done BOOLEAN DEFAULT FALSE,
  created_at    TIMESTAMP DEFAULT NOW()
);

-- 페르소나 지식 메모리 (망각 곡선 적용)
CREATE TABLE persona_memory (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id      UUID REFERENCES personas(id) ON DELETE CASCADE,
  concept         TEXT NOT NULL,           -- 학습한 개념명
  taught_count    INT  DEFAULT 1,          -- 누적 수업 횟수
  stability       FLOAT DEFAULT 1.0,       -- 에빙하우스 안정성 계수 S
  last_taught_at  TIMESTAMP DEFAULT NOW(),
  UNIQUE(persona_id, concept)
);

-- 수업 세션
CREATE TABLE teaching_sessions (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id    UUID REFERENCES personas(id) ON DELETE CASCADE,
  concept       TEXT NOT NULL,
  quality_score INT,                       -- 수업 품질 점수 0~100
  weak_points   JSONB DEFAULT '[]',        -- ["누락 개념1", "누락 개념2"]
  messages      JSONB DEFAULT '[]',        -- [{role, content, timestamp}]
  created_at    TIMESTAMP DEFAULT NOW()
);

-- 배치고사 및 정규 시험
CREATE TABLE exams (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id       UUID REFERENCES personas(id) ON DELETE CASCADE,
  exam_type        TEXT NOT NULL CHECK (exam_type IN ('placement', 'regular')),
  level            INT  NOT NULL,
  questions        JSONB NOT NULL,          -- [{id, type, content, options, answer, concept_tag, difficulty}]
  user_answers     JSONB DEFAULT '[]',
  persona_answers  JSONB DEFAULT '[]',      -- [{question_id, thought, answer}]
  user_score       INT,
  persona_score    INT,
  combined_score   INT,
  passed           BOOLEAN,
  created_at       TIMESTAMP DEFAULT NOW()
);

-- 약점 태그 누적
CREATE TABLE weak_point_tags (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id    UUID REFERENCES personas(id) ON DELETE CASCADE,
  concept       TEXT NOT NULL,
  fail_count    INT  DEFAULT 1,
  last_failed_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(persona_id, concept)
);

-- 인덱스
CREATE INDEX idx_persona_memory_persona ON persona_memory(persona_id);
CREATE INDEX idx_teaching_sessions_persona ON teaching_sessions(persona_id, created_at);
CREATE INDEX idx_exams_persona ON exams(persona_id, created_at);
CREATE INDEX idx_weak_tags_persona ON weak_point_tags(persona_id, fail_count DESC);
```
