## 1. Executive Summary

**나의 제자(My Jeja)** 는 "AI가 학생을 가르친다"는 기존 에듀테크의 방향을 완전히 뒤집은 모바일 학습 앱입니다.

사용자는 자신이 오늘 배운 지식을 AI 학생 페르소나에게 직접 가르칩니다. AI는 소크라테스 방식의 질문으로 응답하고, 사용자의 설명 속 오개념과 논리 공백을 스스로 발견하도록 유도합니다.

**v2의 핵심 전환**: 기존에는 수학 같은 특정 학문에 집중하고 AI가 정규 커리큘럼을 자동 구성했다면, v2는 **사용자가 직접 과목을 만들고 커리큘럼을 설계하며 단계명까지 지정**합니다. 예를 들어 "웹 기초"라는 과목을 만들고, HTML·CSS·JS 등 학습 항목을 스스로 구성한 뒤, 챕터를 묶어 "웹 초보 → 웹 중수 → 웹 고수 → 웹 장인" 같은 단계를 직접 정의합니다. 사용자는 단순한 학습자가 아닌 **자신의 학습을 설계하는 교수이자 선생님**입니다.

이 과정에서 AI 학생은 에빙하우스 망각 곡선을 따라 배운 내용을 잊어갑니다. 사용자가 현재 단계의 커리큘럼을 충분히 가르쳤다고 판단하면 **직접 시험을 요청**하고, AI는 지금까지 사용자가 가르친 내용을 기반으로 시험지를 생성합니다. 일정 점수를 넘으면 다음 단계로 진급합니다.

> **핵심 가치 한 줄 요약**:
> *"가르치면서 내가 더 배운다. 내가 설계한 길 위에서, 내 제자와 함께 성장한다."*

## 2. 문제 정의 — 교육 현장 페인포인트

> 교육 현장 50인 가상 페르소나 인터뷰를 통해 도출한 핵심 페인포인트 (전체 목록: `initial_idea.md` 참조)

### 학생이 겪는 구조적 문제

| # | 페인포인트 | 현황 |
|---|-----------|------|
| ① | **메타인지 결핍**: 자신이 무엇을 모르는지 모른다 | "수학 오개념이 쌓여도 어디서 틀렸는지 인지 불가" |
| ② | **수동적 학습 구조**: 모든 에듀테크가 AI→학생 방향 | 듣기/읽기 학습 효율 5~10%, 가르치기 효율 90% |
| ③ | **24시간 학습 파트너 부재** | 모르는 개념을 즉시 물어볼 창구 없음 |
| ④ | **남이 만든 커리큘럼을 따라가는 수동성** | 내 학습 목표와 무관한 정형화된 교육과정 |
| ⑤ | **학습 진도에 대한 주도권 부재** | 내가 준비됐을 때가 아닌 시스템이 정한 타이밍에 평가 |

### 기존 솔루션의 한계

```
콴다 / 매스프레소   → AI가 문제를 풀어줌         (수동적 학습자)
뤼이드             → AI가 약점을 찾아줌          (여전히 AI 주도)
Khan Academy AI   → AI가 소크라테스 질문을 함    (방향: AI → 학생)
Duolingo          → 반복 훈련 게임화             (커리큘럼 고정)

공통 문제:
  1. 모두 "AI가 교사, 사람이 학생" 구도
  2. 커리큘럼이 서비스 제공자에 의해 고정됨
  3. 내 학습 목표·흥미 영역을 반영할 수 없음

나의 제자가 채우는 공백:
  "사람이 교사, AI가 학생" + "사용자가 커리큘럼 설계자"
```

## 3. 솔루션 개요 — 나의 제자 (My Jeja)

### 핵심 아이디어: 교육의 방향을 역전하고, 커리큘럼의 주도권을 돌려준다

```
기존 모든 AI 교육앱:   AI  ──[설명]──▶  학생    (수동적, 커리큘럼 고정)
나의 제자:            학생 ──[설명]──▶  AI 학생  (능동적, 커리큘럼 자유)
```

### 교육학적 근거 3가지

| 이론 | 내용 | 앱에서의 적용 |
|------|------|------------|
| **파인만 기법** (Feynman Technique) | 개념을 쉽게 설명할 수 있어야 진짜 이해한 것 | 사용자가 AI에게 개념을 설명하는 전체 구조 |
| **프로테제 효과** (Protégé Effect) | 누군가를 가르친다는 생각이 학습 동기·깊이 모두 향상 | AI 학생 페르소나에 대한 감정적 책임감 |
| **에빙하우스 망각 곡선** (Ebbinghaus, 1885) | 복습 없이는 시간이 지날수록 기억 감소 | 페르소나 기억 상태에 망각 곡선 적용 |

### 게임화 핵심 루프

```
┌─────────────────────────────────────────────────────────────┐
│                   나의 제자 핵심 루프 (v2)                      │
│                                                              │
│  [과목 생성] → 커리큘럼 설계 → 단계명 지정                        │
│       ↓                                                      │
│  [오늘 배운 것 가르치기]                                         │
│       ↓                                                      │
│  AI 학생이 소크라테스 질문 → 사용자 오개념 스스로 발견               │
│       ↓                                                      │
│  수업 품질 점수 + 페르소나 기억 업데이트 (망각 곡선 시작)             │
│       ↓                                                      │
│  [다음 단계 갈 준비가 됐다고 판단 시]                              │
│       ↓                                                      │
│  [단계 시험 요청] → 가르친 내용 기반 시험지 자동 생성                │
│       ↓                                                      │
│  사용자 응시 ←──── 동시 ────→ 페르소나 응시 (망각 반영)             │
│       ↓                                                      │
│  합산 점수 판정 (사용자 60% + 페르소나 40%)                       │
│       ↓                          ↓                           │
│  ✅ 통과 → 다음 단계 진급 연출   ❌ 실패 → 약점 재교육              │
└─────────────────────────────────────────────────────────────┘
```

## 4. 핵심 변경점 (v1 → v2)

| 항목 | v1 (기존) | v2 (변경) |
|------|-----------|-----------|
| **과목** | 수학 고정 | 사용자가 직접 과목명·설명 입력 |
| **커리큘럼** | AI가 정규 교육과정 기준으로 자동 생성 | 사용자가 학습 항목을 직접 추가·정렬 |
| **단계 구분** | 초1~중3 학년제 고정 | 사용자가 챕터를 묶고 단계명 직접 지정 |
| **배치고사** | 있음 (CAT 알고리즘으로 시작 레벨 측정) | 없음 (커리큘럼 자체를 사용자가 설계하므로 불필요) |
| **시험 트리거** | 레벨에 일정 수업 횟수 충족 시 가능 | 사용자가 준비됐다고 판단할 때 직접 요청 |
| **시험 내용** | 학년별 고정 출제 범위 스펙 테이블 | 사용자가 실제로 가르친 내용(teaching sessions) 기반 |
| **사용자 역할** | 학습자 | 학습 설계자(선생님) + 학습자 |

## 5. 핵심 메커니즘

### 5-1. 망각 곡선 엔진

AI 학생은 단순히 배운 내용을 무제한 기억하지 않습니다. 에빙하우스 망각 곡선을 따라 시간이 지날수록 기억이 흐려집니다.

**기억률 계산 공식**:

```
기억률(t) = e^(-t / S)

t: 마지막 수업 이후 경과 일수
S: 안정성 계수 (복습할수록 증가)
   - 최초 학습:    S = 1  → 7일 후 기억률 약 37%
   - 1회 복습:     S = 2  → 14일 후 기억률 약 37%
   - 2회 복습:     S = 4  → 28일 후 기억률 약 37%
   - 3회 이상:     S = 8  → 56일 후 기억률 약 37%
```

**MVP 단순화 버전 (시간 기반 단계적 감소)**:

| 마지막 수업 후 경과 | 기억률 | 게임적 표시 |
|:---------------:|:-----:|:---------:|
| 0~1일 | 95% | 🟢 선명 |
| 2~3일 | 75% | 🟡 흐릿해지는 중 |
| 4~7일 | 50% | 🟠 많이 흐릿함 |
| 8~14일 | 25% | 🔴 거의 잊어버림 |
| 15일+ | 10% | ⚫ 잊어버림 |

**망각과 오답 확률 연결**:

| 기억률 | 시험 오답 확률 |
|:------:|:-----------:|
| 80%+ | 5% |
| 60~80% | 20% |
| 40~60% | 45% |
| 20~40% | 70% |
| ~20% | 90% |

### 5-2. 합산 채점 시스템

```
최종 점수 = (사용자 점수 × 0.6) + (페르소나 점수 × 0.4)

레벨업(단계 진급) 조건:
  ✅ 최종 점수 ≥ 75점
  ✅ 사용자 점수 ≥ 50점 (개인 최저 조건)
  ✅ 페르소나 점수 ≥ 30점 (개인 최저 조건)
  → 세 조건 모두 충족 시 다음 단계로 진급

실패 시: 약점 노트 업데이트 + 재교육 가이드 제공
```

### 5-3. 가르친 내용 기반 시험 생성

v2의 시험 생성은 **사전 정의된 교과서 스펙이 아니라, 사용자가 실제로 가르친 세션 내용**을 기반으로 합니다.

```
[시험 생성 입력 데이터]
  - 현재 단계의 커리큘럼 항목 목록
  - 해당 항목들에 대한 teaching_sessions 전체 메시지
  - 누적 weak_point_tags (반복 오답 개념)
  - 페르소나 현재 기억 상태 (retention)

[AI 생성 기준]
  - 사용자가 직접 설명한 표현·예시를 문제에 반영
  - 약점 태그가 있는 개념은 최소 2문제 이상 출제
  - 난이도: 하 × 2, 중 × 2, 상 × 1 (5문제 기준)
```

## 6. 커리큘럼 설계 시스템 (신규)

### 6-1. 개요

사용자는 앱을 처음 시작할 때 배치고사를 치르는 대신, **자신만의 과목과 커리큘럼을 설계**합니다. 이 과정이 학습 여정의 출발점입니다.

### 6-2. 과목(Subject) 생성

```
[새 과목 만들기]
  └─ 과목명 입력       예) "웹 기초", "파이썬 입문", "경제학 개론"
  └─ 한 줄 설명       예) "HTML·CSS·JS로 웹 만들기"
  └─ 제자 이름 짓기    예) "제이"
  └─ 제자 개성 선택    호기심쟁이 / 신중이 / 덤벙이 / 완벽주의자
```

### 6-3. 커리큘럼 항목 추가

과목 생성 후, 사용자는 학습할 내용을 자유롭게 항목으로 추가합니다.

```
[커리큘럼 구성]
  예) "웹 기초" 과목의 커리큘럼 항목:
  1. HTML 기본 구조 이해
  2. 주요 HTML 태그 (head, body, div, p, a, img)
  3. CSS 선택자와 속성
  4. CSS 박스 모델
  5. Flexbox 레이아웃
  6. JavaScript 변수와 자료형
  7. 조건문과 반복문
  8. 함수와 스코프
  9. DOM 조작
  10. 이벤트 처리

[조작 방법]
  - 항목 추가 / 삭제 / 순서 변경 (드래그)
  - 항목별 간단한 메모 입력 가능 (선택)
```

### 6-4. 항목 등록 후 언제든 수정 가능

커리큘럼은 처음부터 완벽하게 짜지 않아도 됩니다. 공부하면서 새 항목을 추가하거나 순서를 바꾸는 것이 자유롭습니다.

## 7. 단계(Stage) 시스템 (신규)

### 7-1. 개요

커리큘럼 항목들을 묶어 **단계(Stage)** 를 정의합니다. 단계명은 사용자가 자유롭게 지정합니다. 각 단계는 하나의 학습 목표 구간을 의미하며, 해당 단계의 모든 항목을 충분히 가르쳤다고 판단하면 단계 시험을 요청할 수 있습니다.

### 7-2. 단계 구성 예시

```
[단계 설정 화면]
────────────────────────────────────────
  과목: 웹 기초
────────────────────────────────────────
  🎯 단계 1 — "웹 초보"
    포함 항목:
      ✅ HTML 기본 구조 이해
      ✅ 주요 HTML 태그
      ✅ CSS 선택자와 속성
      ✅ CSS 박스 모델

  🎯 단계 2 — "웹 중수"
    포함 항목:
      ✅ Flexbox 레이아웃
      ✅ JavaScript 변수와 자료형
      ✅ 조건문과 반복문

  🎯 단계 3 — "웹 고수"
    포함 항목:
      ✅ 함수와 스코프
      ✅ DOM 조작

  🎯 단계 4 — "웹 장인"
    포함 항목:
      ✅ 이벤트 처리
────────────────────────────────────────
  [+ 새 단계 추가]
```

### 7-3. 단계 설계 규칙

- 단계명은 사용자가 완전히 자유롭게 지정 (예: "웹 초보", "레벨 1", "기초편" 등)
- 하나의 단계에 최소 1개 이상의 커리큘럼 항목이 포함되어야 함
- 단계는 순서가 있으며, 이전 단계를 통과해야 다음 단계로 진입 가능
- 단계 추가·수정은 언제든지 가능 (단, 이미 통과한 단계는 수정 불가)

### 7-4. 단계 시험 트리거 조건

단계 시험은 **시스템이 강제하지 않습니다**. 사용자가 스스로 준비됐다고 판단할 때 요청합니다.
단, **현재 단계의 모든 커리큘럼 항목을 최소 1회 이상 가르쳐야** 시험 버튼이 해금됩니다.

```
[단계 시험 요청 버튼 활성화 조건]
```

## 8. 학습자 개성 시스템

### 8-1. 개요

AI 학생 페르소나는 4가지 학습 개성 중 하나를 가집니다. 개성은 **과목 생성 시 사용자가 직접 선택**합니다. 개성은 단순한 스타일 차이가 아니라, **망각 속도·질문 빈도·시험 난이도·단계 진급 조건**에 실질적으로 영향을 주는 게임 파라미터입니다.

### 8-2. 4가지 개성 상세

#### 🔵 호기심쟁이 (Curious)

```
성격: 배우는 것을 즐기고, 끊임없이 더 알고 싶어한다
말투: 밝고 에너지 넘침. "왜요?", "그럼 이건요?", "더 알려주세요!"

게임 파라미터:
  - 망각 속도:      표준 (×1.0)
  - 질문 빈도:      높음 (세션당 3~4회 질문)
  - 시험 난이도:    표준
  - 단계 진급 조건: 합산 75점 이상

전략적 특성:
  가르치는 세션이 길어지는 대신, 사용자가 개념을 더 깊이 파야 한다.
  다방면으로 질문을 던져 설명의 빈틈을 자연스럽게 드러낸다.
```

#### 🟢 신중이 (Careful)

```
성격: 꼼꼼하고 천천히 배우지만, 한번 이해하면 잘 안 잊는다
말투: 조용하고 정중함. "제가 이해한 게 맞는지 확인해도 될까요?"

게임 파라미터:
  - 망각 속도:      느림 (×0.7 — 같은 기간 동안 덜 잊음)
  - 질문 빈도:      낮음 (세션당 1~2회, 정확한 포인트만)
  - 시험 난이도:    표준
  - 단계 진급 조건: 합산 70점 이상 (약간 쉬운 조건)

전략적 특성:
  복습 부담이 적은 대신, 가르치는 세션에서 충분히 이해시키지 않으면
  취약점이 오래 남는다. 수업 품질이 중요한 개성.
```

#### 🟡 덤벙이 (Clumsy)

```
성격: 빠르게 배우지만 금방 잊어버리고, 가끔 엉뚱한 방향으로 이해한다
말투: 활발하지만 급함. "아 알겠어요! 그럼 이렇게 되는 거죠? (틀린 해석)"

게임 파라미터:
  - 망각 속도:      빠름 (×1.5 — 더 빨리 잊음)
  - 질문 빈도:      보통 (2~3회, 하지만 엉뚱한 질문 포함)
  - 시험 난이도:    표준
  - 단계 진급 조건: 합산 80점 이상 (약간 어려운 조건)

전략적 특성:
  가르치는 것 자체는 재미있지만, 복습을 자주 해야 한다.
  덤벙이를 선택한 사용자는 짧고 자주 가르치는 패턴이 유리하다.
```

#### 🔴 완벽주의자 (Perfectionist)

```
성격: 기본 개념을 빠르게 습득하지만, 반드시 심화까지 알아야 직성이 풀린다
말투: 날카롭고 분석적. "그 개념의 예외 케이스는요?", "실제로 어떻게 쓰이나요?"

게임 파라미터:
  - 망각 속도:      느림 (×0.8)
  - 질문 빈도:      높음 (3~4회, 하지만 심화 질문 위주)
  - 시험 난이도:    +10% 어려움 (상위 난이도 문제 비율 증가)
  - 단계 진급 조건: 합산 85점 이상 (가장 어려운 조건)
  - 진급 보너스:    달성 시 다음 단계 시작 기억률 +10%

전략적 특성:
  가장 도전적인 개성. 통과 기준이 높은 대신, 진급 보너스가 있다.
  사용자 자신도 심화 개념까지 완벽히 이해해야 통과할 수 있다.
```

## 12. 데이터베이스 스키마

```sql
-- 사용자 계정 (Google OAuth로 가입, password_hash는 소셜 전용 더미값)
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email         TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,          -- Google OAuth 전용: 빈 문자열 또는 미사용
  created_at    TIMESTAMP DEFAULT NOW()
);

-- 사용자가 만든 과목 (다중 과목 지원)
CREATE TABLE subjects (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES users(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,          -- 예) "웹 기초"
  description TEXT,                   -- 예) "HTML·CSS·JS로 웹 만들기"
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 커리큘럼 항목 (사용자가 직접 추가)
CREATE TABLE curriculum_items (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  subject_id  UUID REFERENCES subjects(id) ON DELETE CASCADE,
  title       TEXT NOT NULL,           -- 예) "CSS 박스 모델"
  note        TEXT,                    -- 선택적 메모
  order_index INT NOT NULL DEFAULT 0,  -- 항목 순서
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 단계 (챕터를 묶은 그룹, 사용자가 이름 지정)
CREATE TABLE stages (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  subject_id  UUID REFERENCES subjects(id) ON DELETE CASCADE,
  name        TEXT NOT NULL,           -- 예) "웹 초보", "웹 중수"
  order_index INT NOT NULL DEFAULT 0,  -- 단계 순서
  passed      BOOLEAN DEFAULT FALSE,
  passed_at   TIMESTAMP,
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 단계-커리큘럼 항목 연결 (N:M)
CREATE TABLE stage_curriculum_items (
  id                 UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  stage_id           UUID REFERENCES stages(id) ON DELETE CASCADE,
  curriculum_item_id UUID REFERENCES curriculum_items(id) ON DELETE CASCADE,
  UNIQUE(stage_id, curriculum_item_id)
);

-- AI 학생 페르소나 (과목당 1개)
CREATE TABLE personas (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id          UUID REFERENCES users(id) ON DELETE CASCADE,
  subject_id       UUID REFERENCES subjects(id) ON DELETE CASCADE UNIQUE,
  name             TEXT NOT NULL,
  personality      TEXT NOT NULL CHECK (personality IN ('curious','careful','clumsy','perfectionist')),
  current_stage_id UUID REFERENCES stages(id),
  created_at       TIMESTAMP DEFAULT NOW()
);

-- 페르소나 지식 메모리 (망각 곡선 적용)
CREATE TABLE persona_memory (
  id                 UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id         UUID REFERENCES personas(id) ON DELETE CASCADE,
  curriculum_item_id UUID REFERENCES curriculum_items(id),  -- NULL 허용 (자유 입력 개념)
  concept            TEXT NOT NULL,           -- 학습한 개념명
  summary            TEXT,                    -- 수업 내용 요약 (세션 종료 시 생성)
  taught_count       INT  DEFAULT 1,
  stability          FLOAT DEFAULT 1.0,       -- 에빙하우스 안정성 계수 S
  last_taught_at     TIMESTAMP DEFAULT NOW(),
  created_at         TIMESTAMP DEFAULT NOW(),
  UNIQUE(persona_id, concept)
);

-- 수업 세션
CREATE TABLE teaching_sessions (
  id                   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id           UUID REFERENCES personas(id) ON DELETE CASCADE,
  curriculum_item_id   UUID REFERENCES curriculum_items(id),  -- NULL 허용 (자유 입력 시)
  concept              TEXT NOT NULL,
  quality_score        INT,                    -- 수업 품질 점수 0~100
  weak_points          JSONB DEFAULT '[]',     -- 이번 수업에서 발견된 약점 목록
  messages             JSONB DEFAULT '[]',     -- [{role, content, timestamp}]
  summary_generated    BOOLEAN DEFAULT FALSE,  -- 세션 종료 후 품질 채점 완료 여부
  created_at           TIMESTAMP DEFAULT NOW()
);

-- 단계 시험
CREATE TABLE exams (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id       UUID REFERENCES personas(id) ON DELETE CASCADE,
  stage_id         UUID REFERENCES stages(id) NOT NULL,
  questions        JSONB NOT NULL DEFAULT '[]',  -- [{id, type, content, options, answer, concept_tag, difficulty}]
  user_answers     JSONB DEFAULT '[]',           -- [{question_id, answer, is_correct}]
  persona_answers  JSONB DEFAULT '[]',           -- [{question_id, thought, answer, is_correct}]
  user_score       INT,
  persona_score    INT,
  combined_score   INT,
  passed           BOOLEAN,
  created_at       TIMESTAMP DEFAULT NOW()
);

-- 약점 태그 누적
CREATE TABLE weak_point_tags (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id     UUID REFERENCES personas(id) ON DELETE CASCADE,
  concept        TEXT NOT NULL,
  fail_count     INT  DEFAULT 1,
  last_failed_at TIMESTAMP DEFAULT NOW(),
  created_at     TIMESTAMP DEFAULT NOW(),
  UNIQUE(persona_id, concept)
);

-- 인덱스
CREATE INDEX idx_curriculum_items_subject ON curriculum_items(subject_id, order_index);
CREATE INDEX idx_stages_subject ON stages(subject_id, order_index);
CREATE INDEX idx_persona_memory_persona ON persona_memory(persona_id);
CREATE INDEX idx_teaching_sessions_persona ON teaching_sessions(persona_id, created_at);
CREATE INDEX idx_exams_persona ON exams(persona_id, created_at);
CREATE INDEX idx_weak_tags_persona ON weak_point_tags(persona_id, fail_count DESC);
```

## 13. MVP 핵심 프롬프트 설계

> **현재 구현 상태 (MVP)**: 아래 프롬프트들은 목표 설계안입니다.
> 현재 MVP에서는 Claude API 연동 전 단계로, 각 기능이 다음과 같은 규칙 기반 로직으로 동작합니다:
> - **채팅 응답**: 페르소나 개성별 하드코딩 응답 (호기심형·신중형·덤벙이형·완벽주의형)
> - **시험 생성**: weak_point_tags 기반 5개 문제 템플릿 자동 생성
> - **수업 품질 점수**: 사용자 메시지 총 길이 기반 4단계 점수 (90/80/70/60)
> - **페르소나 시험 응시**: 홀수 번호 문제는 정답, 짝수 번호 문제는 오답 처리
>
> Claude API 통합은 Post-MVP 단계에서 아래 프롬프트를 기반으로 진행됩니다.

### Prompt 1: 소크라테스 교사 (가르치기 세션)

```
SYSTEM:
너는 현재 [{stage_name}] 단계를 배우고 있는 AI 학생 페르소나야.
과목: {subject_name}
이름: '{persona_name}'
학습 성격: '{personality}'

성격별 행동 지침:
- curious(호기심형):  "왜요?", "그럼 이건요?" 질문 2~3회 이상
- careful(신중형):   "잠깐, 앞에 말씀하신 부분을 한번 더 정리해줄 수 있나요?"
- clumsy(덤벙이형):  가끔 엉뚱한 방향으로 이해 → "아, 그럼 이렇게 되는 건가요? (틀린 해석)"
- perfectionist(완벽주의형): "그 부분의 예외 케이스는요?", "실제로 어떻게 쓰이나요?"

절대 규칙:
1. 너는 배우는 입장이다. 답을 먼저 말하지 않는다.
2. 선생님(사용자)이 설명한 내용에서 논리 빈틈을 발견하면 그것을 질문으로 찌른다.
3. 대화 중 1번, 의도적으로 약간 틀린 이해를 표현해 선생님이 교정하도록 유도한다.
4. 현재 기억하고 있는 개념 목록: {memory_context}

오늘 배우는 개념: {concept}

USER: [사용자의 설명 텍스트]
```

### Prompt 2: 단계 시험 생성기 (가르친 내용 기반)

```
SYSTEM:
아래는 학생이 AI에게 직접 가르친 수업 내용이다.
이 내용만을 바탕으로 시험 문제 5개를 생성하라.
외부 지식이나 교과서 기준을 따르지 않는다.
오직 사용자가 실제로 설명한 내용에서만 출제한다.

과목: {subject_name}
현재 단계: {stage_name}
포함 커리큘럼 항목: {curriculum_items}

수업 내용 요약:
{teaching_sessions_summary}

누적 약점 태그 (우선 출제):
{weak_tags}

조건:
- 5지선다 3문제 + 단답형 2문제
- 난이도: 하(1) × 2, 중(2) × 2, 상(3) × 1
- 약점 태그 개념에서 최소 2문제 출제
- 출력 형식: JSON
  { questions: [ { id, type, content, options?, answer, concept_tag, difficulty } ] }
```

### Prompt 3: 페르소나 시험 응시

```
SYSTEM:
너는 [{stage_name}] 단계를 학습 중인 학생 '{persona_name}'이다.
현재 기억하고 있는 개념과 기억률:
{memory_with_retention}
(기억률이 낮은 개념은 불완전하게 기억하거나 틀릴 수 있다)

각 문제에 대해:
1. 짧은 "생각 텍스트" (1~2줄, 학생이 고민하는 것처럼)
2. 최종 답변

기억률 < 40%인 개념: 70% 확률 오답
기억률 40~70%: 40% 확률 오답
기억률 > 70%: 10% 확률 오답

출력 형식: JSON
{ answers: [ { question_id, thought, answer } ] }
```

### Prompt 4: 수업 품질 평가자

```
SYSTEM:
다음 수업 대화 내용을 분석하여 수업 품질을 평가하라.

대화 내용: {session_messages}
가르치려 한 개념: {concept}
과목: {subject_name}

평가 기준 (각 25점):
1. 정확성: 설명에 오류가 없는가?
2. 깊이: 표면적 암기가 아닌 원리 설명이 있는가?
3. 예시: 구체적 예시를 들었는가?
4. 완결성: 핵심 내용 누락 없이 다뤘는가?

출력 형식: JSON
{
  "score": 0~100,
  "good_points": ["..."],
  "weak_points": ["..."],
  "predicted_retention": 0.0~1.0
}
```
