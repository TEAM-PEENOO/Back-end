# 프롬프트 히스토리 — AI 협업 리포트 (기획 단계 전체)

> 본 문서는 바이브코딩 공모전 'AI 협업 리포트'의 일환으로,
> 기획 단계에서 사용자(태훈)와 AI(Claude)가 주고받은 모든 프롬프트를 기록한 것입니다.

---

## Prompt 01 — 공모전 참여 선언 및 협업 지침 전달

**역할**: 프로젝트 오리엔테이션 / 협업 규칙 설정

```
현재 바이브코딩 공모전 대회에 참가하게 되어서, 너와 내가 협력하여
기획~개발~배포 단계 프로세스까지 함께 진행하게 되었어.
먼저 다음에 관한 사항을 숙지하였음 좋겠네.

심사 기준에 'AI와 협업하는 능력'이 명시되어 있으므로,
단순히 결과물만 제출하지 말고 다음 과정을 'AI 협업 리포트'로 포함하세요.

1. 기획 단계: "LLM과 함께 50개의 교육 현장 페인 포인트를 브레인스토밍하고,
   '실현 가능성-시장성-창의성' 매트릭스를 통해 아이디어를 좁혀가는 과정"을
   프롬프트 로그와 함께 기록.

2. 설계 단계: 아키텍처 설계 시 AI에게 보안 취약점 점검을 요청하거나,
   DB 스키마 최적화를 가이드받은 내용 포함.

3. 지침서(Prompt Engineering): 솔루션 내부에서 사용되는 핵심 프롬프트
   (예: 학생의 수준에 맞게 설명을 변환하는 Prompt)의 고도화 과정을 문서화.

태훈님이 진행 중인 'AI 음성 피싱 가디언'이나 '의류 추천 서비스'에서 쌓은
FastAPI, React-Native 경험을 활용한다면, 2번 아이디어(실시간 보조)가
기술적 완성도를 보여주기에 가장 유리할 것 같습니다.

■ 공모전 주제: AI활용 차세대 교육 솔루션
■ 주요 내용: 교육 현장에서 발생하는 다양한 페인 포인트를 AI 기술로 해결하는 솔루션을 기획 및 구현
■ 가이드라인: 본 공모전은 단순한 온라인 교육 플랫폼(LMS) 구축에 국한되지 않습니다.
  교강사, 수강생, 교육 운영자 등 교육 현장 구성원들의 실질적인 문제를
  날카롭게 해결하는 AI솔루션을 지향합니다.

■ 심사기준
  - 기술적 완성도
  - AI활용 능력 및 효율성
  - 기획력 및 실무 접합성
  - 창의성

※ 본 공모전은 AI와 효율적으로 협업하는 능력을 중요한 심사 항목으로 생각합니다.
  때문에 AI와 기획한 기획문서, 지침서 등의 파일을 프로젝트 내부에 포함시키는 것을 권장 드립니다.

숙지 및 학습이 다 되었다면 다음 지시사항을 알려줄게.
```

**AI 대응 결과**: 공모전 내용 숙지, 메모리 저장 (사용자 프로필 + 프로젝트 컨텍스트)

## Prompt 02 — 기획 단계 시작 (페르소나 + 페인포인트 + 매트릭스)

**역할**: 핵심 아이디어 브레인스토밍 / initial_idea.md 생성

```
자, 다음은, 기획 단계부터 꾸려보자.
너는 지금 교육 기업에서의 CEO, CTO 입장이라고 생각하고,
현재 각자 다른 페르소나 50명을 가상으로 생성하여 진행한다고 생각하자.

그랬을 때, 50개의 교육 현장 페인 포인트를 브레인스토밍하고,
'실현 가능성-시장성-창의성' 매트릭스를 통해 아이디어를 좁혀가는 과정을 진행해줘.

모든 기획 진행-구상 과정은 initial_idea.md 파일에 저장해.
```

**AI 대응 결과**:
- 교육 현장 50인 페르소나 생성 (교사/학생/학부모/운영자/기술/다양성 6개 그룹)
- 50개 페인포인트 도출
- 15개 후보군으로 1차 클러스터링
- F(35%) × M(40%) × C(25%) 가중치 매트릭스 적용
- Top 3: D(AI 개인 튜터) + F(이탈 예측) + N(대형 강의 AI 조교) → D+F 통합안 도출
- `initial_idea.md` 저장

## Prompt 03 — 제약 조건 반영 재기획 (1주일 + 창의성 우선)

**역할**: 개발 제약 조건 하의 아이디어 재평가 / initial_idea_2.md 생성

```
제약사항이 생겼다!
개발 기간은 일주일 내외이며, 빠른 MVP 구성이 가능한 어플리케이션을 만들어야한다는
데드라인과 작업 조건이 생긴 상황.

현재 기간 내에 바이브코딩으로 구현할 수 있는 범위 내로
앱의 구현 난이도를 조절하여 최종 아이디어 도출안을 생성해줘.

물론 이전 initial_idea.md 를 참고해서 진행하는 과정이야.

그리고 추가로 시장에 많은 기획 내용보다도,
창의성있고 참신한(시장에 도출되지 않은) 기획의 내용에 더 많은 점수를 부여할거야.

내가 제안한 조건을 모두 참고해서 작업을 진행하고,
도출안을 initial_idea_2.md 파일에 저장해.
```

**AI 대응 결과**:
- 기존 D+F 통합안 폐기 판정 (시장 포화 + 1주 불가)
- 평가 가중치 재조정: F(35%) × M(20%) × C(45%)
- 교육의 방향을 역전하는 역발상 아이디어 도출
- 최종 선정: **TeachBack AI** ("가르쳐봐") - 학생이 AI에게 가르치는 구조
- 교육학 근거: 파인만 기법 + 프로테제 효과 + 소크라테스 문답법
- `initial_idea_2.md` 저장

## Prompt 04 — 게임화 아이디어 추가 (나만의 학생 키우기)

**역할**: 사용자 주도 아이디어 제안 및 AI 비판적 평가 / surved_initial_idea.md 생성

```
바로 설계로 넘어가기보다 기획을 조금 더 정리해보자.
너가 제시한 아이디어를 바탕으로 새롭게 덧붙일 아이디어가 생각났어.

현재의 제안한 기획서만으로는 학생의 자율성이 부족한 느낌이 들어서.
게임처럼 나만의 학생 키우기 라는 느낌으로 하여.

초등학생 1학년 수준부터 시작하여 대학원생 수준까지 자율적으로 키울 수 있는
페르소나를 생성하여 사용자가 자신이 배운 지식같은 것을 학생에게 가르쳐주듯
계속하여 학습시키고,

사용자는 원할 때 마다 페르소나가 학습한 내용을 바탕으로 실제 그 레벨에 맞는
학생 수준(예- 페르소나가 현재 초등학생 1학년 수준 레벨에 있다면,
시험도 실제 초등학생 1학년 수준 레벨의 시험지로 구성)으로
실제 시험과 동등하게 생성하여 페르소나와 내가 함께 시험을 치뤄

(물론 여기서 페르소나는 매번 학습한 내용이 다 기억나는 것이 아니라,
인간처럼 일정 기간이 지나면 일부분 잊어먹도록 설계하여
정말 실제 사람과 같은 환경으로 진행되도록 하는게 좋겠어),

둘의 점수 합이 일정 점수 이상을 통과하면 내 학생이 다음 학년으로 승급할 수 있는
기능을 가지도록 만들고싶어.

내가 현재 제시한 아이디어가 어떨 것 같은지 평가하고
보완도 할 수 있는 것이 있으면 보완 아이디어도 비판적으로 덧붙여 제시해줬으면 좋겠어.

그리고 최종 결론안을 surved_initial_idea.md 에 저장해줘.
```

**AI 대응 결과**:
- 아이디어 강점 4가지 분석 (게임 루프, 망각 곡선, 감정 투자, 함께 시험)
- 비판적 우려 5가지 지적 + 각 보완안 제시
  - 합산 점수 논리 허점 → 개인 최저 점수 조건 병행
  - 커리큘럼 정합성 → 학년별 출제 범위 스펙 테이블 주입
  - 망각 파라미터 밸런싱 → 에빙하우스 기반 안정성 계수 설계
  - 1주 범위 초과 → 수학 1과목 × 초1~중3 MVP 제한
  - 페르소나 시험 UX → "생각 텍스트" 표시
- 4가지 보완 아이디어 추가 제안 (개성 시스템, 수업 품질 피드백, 졸업식 연출, 약점 노트)
- 솔루션명 확정: **"나의 제자 (My Jeja)"**
- `surved_initial_idea.md` 저장

## Prompt 05 — 기획 마무리 (배치고사 + 개성 시스템 + 최종 기획서)

**역할**: 최종 기능 추가 및 완성 기획서 작성 / proposal.md + prompt_history.md + instruction.md 생성

```
기획 단계의 마지막이야.
마지막으로 추가로 '배치고사' 기능을 넣어서
자신의 수준에 맞게 초기 페르소나를 형성할 수 있는 기능,
학습자 개성시스템을 넣어서
최종적으로 기획서 문서를 작성해서 proposal.md 파일로 저장해줘.

그리고 이때까지 내가했던 프롬프트의 내용들을 prompt_history.md 파일에 모두 저장하고,
추가로 내가 했던 프롬프트 중 "지침서" 역할을 하는 프롬프트 내용을
instruction.md 파일에 저장해줘.
```

**AI 대응 결과**: 본 문서 + `instruction.md` + `proposal.md` 생성
